## 🎯 分工原则 (以 Gemini 为核心)

在 Phase 5 中，**Gemini 将作为核心全栈开发主力**，负责绝大多数的 UI/UX 设计、业务逻辑实现以及前后端集成工作。**Claude 将转为辅助角色**，专注在代码审查、极端测试、算法微调以及安全评估上。

| AI | 角色定位 | 负责内容 |
|-------|----------|----------|
| **Gemini** | **主导开发 (Primary)** | UI/UX 设计、React 组件开发、Context API 业务逻辑、系统集成、动效实现 |
| **Claude** | **辅助支持 (Auxiliary)** | 复杂算法优化、边缘边界测试 (`TDD`)、Prompt 工程调优、安全与代码审查 |

## 🚀 推荐执行机制

### 同步推进流（最优化）

**Gemini 作为推进主体，在每一个微型 Sprint 中执行如下策略：**
1. **Gemini 发起**：完成基础结构的组件建构。
2. **Gemini 填充**：实现所有数据流、交互动画与 React 纯逻辑。
3. **Gemini `Merge` 前请教**：当遇到特定难点（如高复杂度的 Prompt 规划或 PDF 内嵌字库问题），移交 `task_request` 给 Claude。
4. **Claude 交付**：提交代码片段或纯测试报告。
5. **Gemini 关单**：全场景确认后封闭当前业务点。
